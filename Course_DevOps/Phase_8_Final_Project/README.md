## Phase 8 — Final Project

**Недели 20–21** | Микросервисное приложение в облаке

Итоговый проект курса: развёртывание микросервисного приложения с полным стеком — инфраструктура, Kubernetes, CI/CD, мониторинг, логирование, безопасность.

### Цель

Продемонстрировать навыки Middle DevOps: от Terraform до мониторинга, с GitOps и security.

### Структура

| Файл | Описание |
|------|----------|
| `project_spec.md` | Спецификация проекта, требования, acceptance criteria |
| `architecture_diagram.md` | Mermaid-диаграмма архитектуры |
| `checklist.md` | Чек-лист для самопроверки перед сдачей |

### Порядок выполнения

1. Изучите `project_spec.md` и `architecture_diagram.md`.
2. Спроектируйте и реализуйте инфраструктуру (Terraform).
3. Разверните приложение в Kubernetes (Helm, Argo CD).
4. Настройте CI/CD (build, push, GitOps).
5. Добавьте мониторинг (Prometheus, Grafana) и логирование (Loki).
6. Внедрите security (Trivy, secrets, securityContext).
7. Пройдите `checklist.md` перед завершением.

### Оценка времени

**30–40 часов** (2–3 недели при 10–15 ч/нед).
