# Подготовка к собеседованию — Middle DevOps

50 вопросов с ответами для самопроверки. Ориентир — уровень Middle DevOps.

---

## Linux и сеть

1. **Как найти процессы, слушающие порт 8080?**  
   `ss -tlnp | grep 8080` или `lsof -i :8080`.

2. **Что такое inode?**  
   Структура метаданных файла в файловой системе (размер, права, владелец, ссылки). Каждый файл имеет уникальный inode.

3. **Разница между `kill` и `kill -9`?**  
   `kill` отправляет SIGTERM (корректное завершение); `kill -9` — SIGKILL (принудительное, не перехватывается).

4. **Как перенаправить stderr в stdout?**  
   `command 2>&1` или `command &>`.

5. **Что такое load average?**  
   Среднее количество процессов в состоянии runnable + uninterruptible sleep за 1, 5, 15 минут. Показывает загрузку системы.

## Git

6. **Как откатить последний коммит, сохранив изменения?**  
   `git reset --soft HEAD~1`.

7. **Разница между merge и rebase?**  
   Merge создаёт merge commit; rebase переписывает историю, применяя коммиты поверх другой ветки.

8. **Как разрешить конфликт слияния?**  
   Открыть файл, убрать маркеры `<<<<<<<`, `=======`, `>>>>>>>`, оставить нужный код, `git add`, `git commit`.

9. **Что такое git stash?**  
   Временное сохранение незакоммиченных изменений в стек, чтобы переключиться на другую ветку.

10. **Как посмотреть, кто изменил строку в файле?**  
    `git blame <file>`.

## Docker

11. **Разница между COPY и ADD в Dockerfile?**  
    COPY — только копирование; ADD — копирование + распаковка архивов + загрузка по URL.

12. **Что такое multi-stage build?**  
    Несколько этапов `FROM` в одном Dockerfile; итоговый образ содержит только артефакты последнего этапа.

13. **Как ограничить память контейнера?**  
    `docker run -m 512m` или в compose: `mem_limit: 512m`.

14. **Разница между volume и bind mount?**  
    Volume — управляется Docker, хранится в `/var/lib/docker/volumes`; bind mount — привязка к конкретному пути на хосте.

15. **Что такое .dockerignore?**  
    Исключает файлы из контекста сборки (аналог .gitignore для `docker build`).

## Kubernetes

16. **Разница между Deployment и StatefulSet?**  
    Deployment — stateless, произвольные имена подов; StatefulSet — stateful, стабильные имена и порядок.

17. **Что такое readiness и liveness probe?**  
    Readiness — готов ли под принимать трафик; liveness — жив ли контейнер (перезапуск при fail).

18. **Как Pod получает трафик от Service?**  
    kube-proxy настраивает iptables/ipvs правила; Service — ClusterIP, селектор по labels, endpoints из подов.

19. **Что такое ConfigMap и Secret?**  
    ConfigMap — конфигурация (не секретная); Secret — чувствительные данные (base64, можно шифровать at rest).

20. **Разница между Job и CronJob?**  
    Job — одноразовое выполнение; CronJob — по расписанию.

## Helm

21. **Что такое Helm chart?**  
    Пакет манифестов K8s с шаблонами (Go template) и values.

22. **Что такое Helm release?**  
    Установленный экземпляр чарта в кластере с уникальным именем.

23. **Приоритет values?**  
    `--set` > `-f file` > `values.yaml`.

24. **Для чего helm template?**  
    Рендерит шаблоны в итоговые YAML без установки (проверка, отладка).

25. **Как откатить релиз?**  
    `helm rollback <release> <revision>`.

## Terraform

26. **Что такое Terraform state?**  
    Файл/хранилище с текущим состоянием инфраструктуры; Terraform сравнивает его с желаемым и планирует изменения.

27. **Зачем remote backend?**  
    Совместная работа команды, блокировки, единое хранилище state, защита от потери.

28. **Разница между resource и data source?**  
    Resource — создаёт/управляет; data source — только читает существующие ресурсы.

29. **Что делает terraform destroy?**  
    Удаляет все ресурсы, управляемые Terraform в текущем state.

30. **Зачем модули?**  
    Переиспользование, DRY, стандартизация (VPC, compute, storage).

## Ansible

31. **Что такое идемпотентность?**  
    Повторный запуск не меняет уже настроенное состояние; changed=0 при отсутствии изменений.

32. **Разница между task и handler?**  
    Task выполняется при каждом запуске (если changed); handler — только при notify и changed, в конце play.

33. **Что такое Ansible Vault?**  
    Шифрование переменных (пароли, ключи) для хранения в Git.

34. **Для чего group_vars?**  
    Переменные для группы хостов (например, web, db).

35. **Что делает --check?**  
    Dry-run: показывает изменения без применения.

## CI/CD

36. **Разница между CI и CD?**  
    CI — сборка, тесты при коммите; CD — доставка/деплой до прода.

37. **Что такое GitOps?**  
    Истина в Git; CD-система (Argo CD) синхронизирует кластер с репозиторием.

38. **Что делает Argo CD?**  
    Синхронизирует состояние K8s с Git (манифесты, Helm charts); поддерживает prune, selfHeal.

39. **Что такое drift?**  
    Расхождение между состоянием в Git и в кластере (ручные изменения).

40. **Зачем security gate в CI?**  
    Блокировать merge/deploy при критичных уязвимостях (Trivy, SAST).

## Мониторинг

41. **Типы метрик Prometheus?**  
    counter, gauge, histogram, summary.

42. **Что делает rate()?**  
    Скорость прироста метрики (per second) за указанный интервал.

43. **Для чего Alertmanager?**  
    Маршрутизация, группировка, отправка уведомлений об алертах (Slack, PagerDuty, email).

44. **Чем Loki отличается от Elasticsearch?**  
    Loki использует лейблы вместо полного индекса; дешёвое хранение, быстрые запросы по лейблам.

45. **Что такое Promtail?**  
    Агент для сбора логов и отправки в Loki.

## Безопасность

46. **Зачем runAsNonRoot?**  
    Запрещает запуск контейнера от root; снижает риски при компрометации.

47. **Что такое SealedSecrets?**  
    Зашифрованная версия K8s Secret для хранения в Git; controller расшифровывает в кластере.

48. **Для чего SOPS?**  
    Шифрование YAML/JSON файлов с секретами (PGP, KMS) для хранения в Git.

49. **Что делает Trivy?**  
    Сканирование образов, файловой системы, конфигов на уязвимости (CVE).

50. **Зачем capabilities: drop ALL?**  
    Удаляет все Linux capabilities у контейнера; минимальные привилегии.
